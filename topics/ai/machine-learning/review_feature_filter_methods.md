# 一：研究動機與核心理論

1. IoT與自動化技術進步導致資料變得極度複雜，特徵過多會導致在空間上變得稀疏，需要去除冗餘、不相關等的雜訊，避免預測失準，進而提升效能，避免過擬合、節省算力、增強可解釋性。
2. 特徵選擇分三大流派，Filter直接拿初始篩選，將剩餘特徵都交給決策樹分類；Wrapper一直嘗試不同組合交給決策樹，得出結果後再換下一組；Embedded直接在決策樹中忽視不需要的特徵。因為filter 可快速移除變數的特性，此研究使用過濾法
3. 為了根據學生群體分配「需要的教育資源多寡」，使用三個分類等級對中學生成績資料集做分類預測，分別做不同管理方式「急需介入」、「需要監控」、「可最少關注」
4. 因為決策樹可處理數值與類別型資料，且規則易於理解、過往文獻中表現較佳，選擇其作為預測模型，透過遞歸分割，不斷切割資料直到分類純度足夠高
# 二：四大過濾法與模型機制

## 壹、四大特徵過濾法
1. ANOVA: F=組間差異(MSB)/組內誤差(MSE)，適用於特徵是數值型且預測目標是類別型，如缺席天數(連續數字)對應成績等級
2. Kendall: T=2*(一致對數n_c-不一致對數n_d)/樣本數n*(樣本數n-1)，適用於樣本數較小且須評估排序相關性，如自由時間(等級排序)與成績等級的相關性
3. 卡方檢定: $x_{c}^{2}=\sum\frac{(觀察值頻率O_{i}-期望值頻率E_{i})^{2}}{E_i}$，適用於類別型輸入對應類別型目標，如家裡是否有網路(純類別標籤)對應成績等級
4. 互資訊: $I(X;Y)=混亂度H(X)-混亂度H(X|Y)$，適用於非線性關係與非常態分配等類別型對應，如父母同居狀態對應成績等級
## 貳、決策樹演算法
1. ID3: 資訊增益最高
2. C4.5: 資訊增益率，並且懲罰類別分支過多的特徵
3. CART: $Gini(D) = 1 - \sum p_i^2$，強制生成二元樹，尋找讓切分後Gini不純度降到最低的閾值
# 三：實驗設計與資料工程

## 壹、資料翻譯
1. Label 編碼: 將類別直接映射到整數數字上，適用於只有兩種選擇或有順序大小關係的特徵，如性別
2. One-hot 編碼: 將特徵拆解成多個獨立的二元特徵，適用於沒有順序大小關係的特徵，如選校原因
## 貳、錄取線
1. 設定三個門檻，90%信賴區間、95%信賴區間、99%信賴區間，分別以kendall 分析的特徵數量為比較基準(16, 12, 6)，比較各特徵過濾法的表現
### 參、防作弊
1. 80%訓練，20%測試，避免Information Leakage
2. 訓練時採用10折交叉驗證，也就是先平均分十份，九份訓練，一份驗證，重複十次分別讓每份都當過驗證資料集，降低過擬合風險
# 四：數據實證與最終結論

## 壹、三大核心指標
1. 準確率: 模型的正確數/總數
2. 精確率: 真的不及格數/模型預測不及格的數
3. 召回率: 模型預測不及格數/所有不及格數
## 貳、三種門檻結果
1. 90%(16個): Kendall透過排序同步率，保留夠多有用資訊，準確率86%，ANOVA墊底80%
2. 95%(12個): 互資訊的透過混亂度比較，抓到非線性關係，準確率0.87%
3. 99%(6個): 太少特徵選擇導致泛化性不足，結果都在81%82%左右，不使用任何特徵選擇反而準確率85%
## 參、實際應用
1. 拯救落後學生(找level 0): 使用95%門檻的MI，預測落後學生的精確率與召回率高達92%
2. 尋找頂尖學生(找level 2): 因為資料集中優秀的學生極少(樣本不平衡)，沒有足夠的範例可以學習，召回率只有70%，容易把天才誤認為一般人

# 五、感想
疑問：
1. IoT導致的資訊複雜是否真的影響到學生特徵的蒐集
2. 決策樹本身是embedded，為何論文說明此次使用的是filter
3. 此研究以kendall 95%信賴區間的特徵數為基底，但如果kendall 本身就對某種資料(像純類別資料)有盲點，而採取其只能留12個特徵的「尺」來評判其餘三個方法量出來也會是歪的，應該直接硬性規定三局的特徵量避免偏好單一機制
